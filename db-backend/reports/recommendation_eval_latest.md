## 评估协议说明

- 采用用户级时间切分：按评分、喜欢、想看行为的时间顺序切分历史窗口与未来窗口。
- 历史窗口用于构建用户画像，画像权重与线上推荐保持一致。
- 未来窗口仅将 `rating >= 4.0` 与 `like` 视为主相关真值，`want_to_watch` 仅作为辅助行为。

## 严格协议附加说明

- 协同过滤类算法仅使用严格早于切分时点的历史行为快照，不再读取当前整库的完整交互。
- TF-IDF、图内容和 PPR 仅使用历史画像与静态电影元数据/知识图谱结构。
- CFKG 在严格协议下改为“历史正反馈电影合成用户表示 + 候选集 learned reranker 精排”，避免直接使用现成用户 embedding。
- 若需要对 CFKG 做完全无泄漏评估，仍需使用同一时间快照重导数据并重训嵌入模型。

## 算法分析

### CFKG

- 定位: 当前线上默认链路，联合用户行为与知识图谱嵌入。
- 优点: 同时建模交互关系和知识图谱结构，适合作为主推荐链路。
- 局限: 训练与部署成本更高，实验复现实验链路更复杂。
- 本次结果: Precision@10=0.0820, Recall@10=0.4330, NDCG@10=0.2709, Coverage=0.0169, Diversity=0.7648

### ItemCF

- 定位: 传统物品协同过滤，对照用户行为共现关系。
- 优点: 对强历史共现信号响应稳定，实现简单，适合作为行为基线。
- 局限: 冷启动与长尾覆盖较弱，缺少图谱语义与多跳关系建模。
- 本次结果: Precision@10=0.0800, Recall@10=0.4030, NDCG@10=0.2294, Coverage=0.0123, Diversity=0.8552

### Hybrid

- 定位: 融合 CF、Content、PPR 的多策略推荐。
- 优点: 兼顾稳定性、解释性与图谱多跳能力，整体表现通常较稳。
- 局限: 调参与系统复杂度更高，单条解释不如纯算法直接。
- 本次结果: Precision@10=0.0780, Recall@10=0.3747, NDCG@10=0.2291, Coverage=0.0160, Diversity=0.8492

### Graph CF

- 定位: 基于用户-电影评分图的协同过滤。
- 优点: 能利用相似用户的图邻域行为，效果通常优于纯 ItemCF。
- 局限: 仍然依赖足够行为数据，对实体语义建模较弱。
- 本次结果: Precision@10=0.0700, Recall@10=0.3163, NDCG@10=0.1869, Coverage=0.0074, Diversity=0.7868

## 推荐结论

- 在当前协议下，综合 NDCG@10 和 Recall@10 表现最优的算法为 `CFKG`。
- `CFKG` 仍建议作为线上主推荐链路，传统基线主要承担论文对照角色。
- 严格协议下的 `CFKG` 采用历史正反馈合成用户表示并复用 learned reranker，不等价于按同一时间快照完整重训后的线上模型。
- `ItemCF` 与 `TF-IDF` 可分别作为行为基线与纯内容基线，便于说明知识图谱算法的增益。
