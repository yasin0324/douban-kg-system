## 1. 核心成果：企业级分布式爬虫系统

我们成功构建了一个高可用、抗封禁、自动循环的爬虫系统。

### 1.1 核心能力

- **抗封禁体系 (Anti-Scraping Hardening)**:
    - **身份伪装**: 每个 Batch (500部) 自动轮换全新的现代化 User-Agent 和随机 BID Cookie。
    - **浏览器隔离**: 使用 Playwright 的 Context 隔离机制，确保每次任务都在干净的环境中执行。
    - **智能调度器 (Global Scheduler)**:
        - **小憩**: 每爬取 200 条，强制休息 3 分钟。
        - **长睡**: 每运行 4 小时，强制休眠 45 分钟 (模拟人类作息)。
- **混合发现引擎 (Hybrid Discovery)**:
    - **主动探索 (`discover_movies.py`)**: 利用 API 主动扫描热门、高分、冷门等标签，快速扩充种子库。
    - **自我造血 (`crawl_movie.py`)**: 在爬取详情页时，自动提取“喜欢这部电影的人也喜欢”列表中的 ID，实现 BFS 滚雪球式增长。
- **永动服务架构**:
    - 主程序采用 `While True` 循环模式：`领任务 -> 执行 -> 休息 -> 再领任务`。
    - 即使数据库暂时没任务，也会自动待机等待新种子，全程无需人工干预。

### 1.2 基础设施

- **数据库**: MySQL
    - `subjects`: 种子库 (存放待爬 ID)
    - `movies`: 数据仓 (存放完整电影信息)
- **监控**: `monitor.py` 提供实时进度条、速度统计和状态监控。

## 2. 运行指南

### 启动命令 (使用 uv)

请在三个独立的终端窗口中运行：

**窗口 1：监控面板**

```bash
uv run scripts/monitor.py
```

**窗口 2：主爬虫 (核心服务)**

```bash
uv run crawl_movie.py --workers 4
```

**窗口 3：种子搜寻 (辅助)**

```bash
uv run discover_movies.py
```

## 3. 后续规划 (Phase 2: 知识图谱与应用)

下一阶段我们将利用采集到的数据进行核心业务价值开发。

### 3.1 知识图谱构建 (Neo4j)

- **Schema 设计**: 定义 `Movie` (电影), `Person` (影人), `Genre` (类型) 等节点及其关系。
- **数据清洗与迁移 (ETL)**: 开发 `build_graph.py` 将 MySQL 中的半结构化数据清洗并导入 Neo4j。
- **图谱可视化**: 在 Neo4j Browser 中验证节点连接性。

### 3.2 推荐算法实现

- **基于内容的推荐**: 利用 Tags 和 Genre 计算电影相似度。
- **基于图的推荐**: 利用 Graph Embedding 或简单的多跳查询 (如 "同一个导演+同一个演员") 挖掘潜在关联。

### 3.3 Web 应用开发
