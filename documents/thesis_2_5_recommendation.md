# 2.5 推荐系统设计与实现

## 2.5.1 设计目标

本系统的推荐模块建立在前文构建完成的豆瓣电影知识图谱之上，目标是实现一个兼具个性化、可解释性和实验可对比性的电影推荐系统。与开题阶段“至少实现两种知识图谱推荐算法”的要求相比，当前实现进一步扩展为“线上主链路 + 多算法实验对照”的结构：

- 线上默认推荐链路采用 `CFKG`
- 研究对照层保留 `ItemCF / TF-IDF / CF / Content / PPR / Hybrid / CFKG`
- 前端默认只暴露 `CFKG`，避免把论文对照算法直接混入产品主体验

## 2.5.2 总体架构

推荐系统采用“用户画像构建 + 候选召回 + 排序融合 + 可解释展示”的总体架构：

1. 从用户评分、喜欢、想看行为中构建推荐画像。
2. 将画像映射为电影、类型、导演、演员、地区、语言等特征。
3. 使用不同推荐算法生成候选结果。
4. 统一返回推荐列表、推荐理由、画像标签和解释图数据。

其中，用户画像不再依赖前端显式指定种子电影，而是由后端基于完整行为自动生成，这使推荐系统更符合实际应用场景。

## 2.5.3 基线模型实现

为了满足论文实验中的标准对照需求，本系统实现了两类传统推荐基线：

### 1. ItemCF

`ItemCF` 仅基于用户正向行为的物品共现关系进行推荐。系统将用户历史高分、喜欢和想看行为视为正向交互，通过计算电影之间的加权余弦相似度，得到候选电影及相似度得分。该方法不依赖知识图谱，仅作为传统协同过滤基线。

### 2. TF-IDF

`TF-IDF` 仅基于电影内容文本和元数据进行推荐。电影文档由标题、类型、地区、语言、导演、演员和剧情简介等字段构成，采用 TF-IDF 向量表示电影内容，再通过用户历史正向电影的加权平均向量构建用户兴趣向量，并使用余弦相似度完成候选排序。该方法不利用知识图谱结构，仅作为纯内容推荐基线。

## 2.5.4 基于知识图谱的核心算法实现

在知识图谱推荐方面，系统实现了多种核心算法：

### 1. Graph Content

该算法直接利用知识图谱中的实体关系，根据用户画像中的高权重类型、导演、演员等实体进行候选召回，再结合地区、语言、年代和评分接近度完成重排。该算法解释性较强。

### 2. PPR

该算法以用户画像中的上下文电影为起点，在电影-影人-类型异构图上执行 Personalized PageRank，从而发现多跳隐性关联电影，适合体现知识图谱的结构优势。

### 3. Graph CF

该算法基于 Neo4j 中的 `User-RATED-Movie` 评分图寻找相似用户邻域，并结合弱负反馈信号进行降权。虽然本质上仍属于协同过滤，但其邻域发现过程直接依赖图数据库中的关系结构。

### 4. Hybrid

`Hybrid` 对 `CF / Content / PPR` 三类图谱算法进行动态融合，兼顾稳定性、解释性和隐性关系发现能力。

### 5. CFKG

`CFKG` 是当前系统的默认线上主链路。该方法先结合图协同过滤和图内容推荐获得候选集合，再利用知识图谱嵌入模型进行排序，从而同时建模用户行为关系与知识图谱结构信息。相较于单一算法，`CFKG` 在相关性和综合表现上更适合作为最终线上推荐方案。

## 2.5.5 用户画像与可解释机制

系统将用户评分、喜欢、想看三类行为统一纳入画像构建，并赋予不同权重：

- 评分 `4.0-5.0` 视为强正反馈
- 评分 `3.5` 视为弱正反馈
- 评分 `3.0` 及以下视为弱负反馈
- `like` 视为高权重正反馈
- `want_to_watch` 同时提供弱正反馈和轻探索信号

推荐结果返回时，系统会同时输出：

- 用户画像标签
- 推荐理由
- 代表兴趣电影
- 图谱证据链或基线解释信号

因此，推荐结果不仅能回答“推荐了什么”，还能回答“为什么推荐”。

## 2.5.6 效果评估与实验协议

为保证评估协议与线上实现一致，离线实验采用“用户画像驱动 + 时间切分”的方式：

1. 按时间顺序对用户历史行为进行切分。
2. 历史窗口用于构建用户画像。
3. 未来窗口中 `rating >= 4.0` 和 `like` 作为主相关真值。
4. 统一比较 `ItemCF / TF-IDF / CF / Content / PPR / Hybrid / CFKG` 七种算法。

实验指标包括：

- `Precision@10`
- `Recall@10`
- `NDCG@10`
- `Coverage`
- `User Coverage`
- `Diversity`

系统实现了自动化离线评估脚本，并可输出 JSON 和 Markdown 格式的实验报告，便于论文直接引用。

## 2.5.7 与开题计划相比的变化说明

与开题报告相比，推荐系统的实际实现有以下变化：

1. 原计划强调“至少两种基于知识图谱的推荐算法”，当前实际已扩展为多算法框架，不仅实现了 `Content / PPR / Hybrid / CFKG` 等知识图谱方法，还补充了 `ItemCF / TF-IDF` 两类传统基线。
2. 原计划中的推荐流程更偏“实验接口”，当前系统已经演化为完整的 Web 产品链路，采用后端自动画像驱动，不再要求用户显式选择种子电影。
3. 原计划未明确默认线上算法，当前实现中 `CFKG` 已成为默认主推荐链路，其余算法主要承担实验对照和论文分析任务。

因此，当前推荐模块不仅完成了开题阶段提出的基本目标，而且在实验完整性、系统可演示性和论文可写性方面都有进一步扩展。
